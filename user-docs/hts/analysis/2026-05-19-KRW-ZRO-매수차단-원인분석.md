## 5. 두 단계 차단 흐름

### Phase A — 03:22 ~ 04:43 (NO_SIGNAL, 약 1시간 20분)

- 매도 직후 가격 회복: 1,898 → 1,944 (+2.4%)
- 차트 십자선(04:01) 시점 1,920, 04:22 신고가 1,938 도달
- **그러나 EMA(10) Fast가 EMA(30) Slow를 위로 돌파(Golden Cross)하지 못함**
- STOP_LOSS 직후 두 EMA 모두 하향 정렬 상태였고, 단기 회복으론 GC 재형성에 시간 부족
- `failed_keys=["NO_SIGNAL"]`, `notes="Golden | NO_SIGNAL | bar=596~614"`

### Phase B — 04:44 ~ 14:52 (SURGE_FILTER, 약 10시간)

- 04:44부터 가격 1,949원 이상, GC가 재형성될 만한 시점에 **Surge Filter가 발동**
- 차단 로직 — [core/filters/buy_filters.py:60-66](../../../core/filters/buy_filters.py)
  ```python
  surge_pct = (bar.close - ema_slow) / ema_slow
  if surge_pct > self.threshold_pct:  # 0.02 = 2%
      return FilterResult(should_block=True, reason="SURGE_FILTER", ...)
  ```
- Slow EMA가 아직 따라오지 못한 상태에서 가격이 +2% 이상 이격 → 추격매수 방지로 차단
- 이후 가격 안정화 + Slow EMA 추격 → 14:52에 비로소 통과하여 EMA_GC 매수

## 6. 활성 설정 (mcmax33, EMA 전략)

`mcmax33_latest_params_EMA.json` 핵심값:

```json
{
  "strategy_type": "EMA",
  "fast_period": 10,
  "slow_period": 30,
  "stop_loss": 0.005,
  "take_profit": 0.005,
  "ema_surge_filter_enabled": true,
  "ema_surge_threshold_pct": 0.02,
  "min_holding_period": 1

```

| 파라미터 | 값 | 영향 |
|---|---|---|
| `stop_loss` | 0.005 (0.5%) | 매우 타이트 — 일반 변동성에 즉시 손절 |
| `take_profit` | 0.005 (0.5%) | TS 활성화 트리거 비율 |
| `ema_surge_threshold_pct` | 0.02 (2%) | Slow EMA 대비 2% 이상 이격 시 매수 차단 |
| `min_holding_period` | 1봉 | 보유 1봉 후 즉시 매도 가능 |

## 7. 결론

**봇은 설정대로 정상 동작했다.** 시스템 오류 아님.

매수 차단의 본질적 원인:
1. **`stop_loss=0.5%`가 너무 타이트** → 일반적 변동성으로 즉시 손절 발생
2. 손절 직후 1시간 20분간은 **EMA(10/30) 구조상 Golden Cross 재형성 불가**
3. GC가 재형성될 시점엔 이미 **가격이 Slow EMA 대비 2% 이상 이격** → SURGE_FILTER가 추격매수 차단 (의도된 안전장치)
4. 가격 안정화 후 14:52에 비로소 매수, 단 +5.79% 비싼 가격에 재진입

## 8. 검토 옵션 (사용자 판단용)

| 옵션 | 변경 | 트레이드오프 |
|---|---|---|
| A | `stop_loss` 완화 (0.5% → 1.0~1.5%) | 잦은 손절 감소 / 큰 손실 위험 증가 |
| B | `ema_surge_threshold_pct` 완화 (2% → 3~5%) | 손절 후 빠른 재매수 가능 / 급등 추격매수 위험 증가 |
