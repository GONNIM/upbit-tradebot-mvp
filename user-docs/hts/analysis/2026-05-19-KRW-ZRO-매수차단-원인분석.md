## 2. 이미지 정밀 분석

### 2-1. 차트 화면 구성

- 거래소 차트 (Upbit 모바일 앱 추정), 1분봉
- 좌측 상단 인디케이터 텍스트: `TI+MA  20  200  10` (또는 유사) — **3개 이동평균 표시**
- 라인 색깔:
  - **청록색**: 가장 빠르게 움직임 → EMA 10
  - **노란색**: 중간 속도 → **EMA 20** (또는 25)으로 추정
  - **흰색**: 가장 평평 → EMA 200 (장기)
- 봉정보(04:01 KST): 시 1,913 / 고 1,920 / 저 1,913 / 종 1,920 (**+1.05% 양봉**)

### 2-2. 04:01 시점 시각 관찰

- 차트 십자선이 04:01에 위치
- 십자선 부근에서 **청록색(EMA 10)이 노란색(EMA 20?)을 위로 분리**하는 형태
- 그 후 04:22까지 가격 1,938 신고가, 청록색과 노란색이 점점 벌어짐
- → **사용자가 "Golden Cross"라고 인식한 시점 정확**: 시각적으로는 두 라인의 교차 명확

### 2-3. 결정적 의문

차트의 노란선이 **EMA 20 (또는 25)**이라면 봇의 EMA 30과 기간이 다르다.
EMA 20과 EMA 30은 가격 변동에 대한 반응 속도가 다르므로 **Cross 시점도 다를 수밖에 없다**.

## 3. 봇의 Cross 정의 (코드 근거)

[core/strategy_incremental.py:640-645](../../../core/strategy_incremental.py):

```python
ema_golden_cross = (
    prev_ema_fast is not None
    and prev_ema_slow is not None
    and prev_ema_fast <= prev_ema_slow    # ← 직전 봉: fast가 slow보다 작거나 같음 (Dead)
    and ema_fast > ema_slow                # ← 현재 봉: fast가 slow보다 큼 (Golden)
)
```

**핵심**: Golden Cross는 "현재 정렬"이 아니라 **봉 단위 전환 이벤트** (Dead → Golden 돌파). 두 라인이 시각적으로 가까이 있다가 멀어지는 것만으로는 Cross 아님.

또한 `checks.cross_status` 필드는 "현재 봉의 fast vs slow 정렬"만 기록한다 — **Cross 이벤트 발생 여부와 다른 개념**.

검증: **14:51 BUY_SIGNAL 시점도 `cross_status=Golden`** (즉 BUY는 cross_status만으로는 판정 안 됨, 전환 이벤트가 추가로 필요).

## 5. 04:01 "시각적 Cross"의 정체

### 5-1. 봉 단위 데이터로 본 04:01

- **04:00 봉**: gap = +0.069 → fast (1904.64) > slow (1904.57). **이미 Golden 정렬** (아주 작은 차이지만 fast가 위)
- **04:01 봉**: gap = +3.868 → fast (1908.68) > slow (1904.81). **여전히 Golden**

봇 Cross 조건 `prev_fast ≤ prev_slow AND curr_fast > curr_slow`:
- prev(04:00) fast > slow → **prev 조건 미충족** (Golden이므로 ≤ 아님)
- 따라서 **봇 Cross 이벤트 미발생 → NO_BUY_SIGNAL**

### 5-2. 왜 시각적으로는 Cross처럼 보이는가

1. 03:25 ~ 04:00 사이 약 35분간 박스권(1,896~1,900)에서 EMA 10과 EMA 30이 점진적으로 수렴 → gap 최저 +0.069 (거의 같은 값)
2. **두 라인이 시각적으로 거의 겹친 상태**가 됨
3. 04:01에 +1.05% 양봉 발생 → EMA 10이 한 봉에서 +4원 점프 → 시각적으로 **두 선이 분리되며 청록이 노란을 "돌파하는 것처럼" 보임**
4. 그러나 봉 종가 기준으로는 fast가 단 한 번도 slow 아래로 떨어지지 않았기 때문에 봇 Cross 조건 미충족

### 5-3. 차트 EMA 기간과 봇 EMA 기간의 차이 가능성

만약 차트의 노란선이 **EMA 20** (또는 EMA 25)이라면:
- EMA 20은 EMA 30보다 가격 변동에 더 빠르게 반응
- 매도 직전 가격 1,898 하락 시 EMA 20이 EMA 30보다 더 빠르게 떨어짐
- → **차트 EMA 20에서는 Dead Cross 발생, 봇 EMA 30에서는 미발생** 가능
- → 04:01 양봉 시 EMA 20이 EMA 10에 의해 돌파됨 (차트 시각적 Cross), 봇 EMA 30은 미돌파

**이 가설이 사실이라면 차트 차트와 봇은 서로 다른 시점을 보고 있는 셈.**

→ **확인 필요사항**: 사용자가 사용하는 차트 앱의 정확한 EMA 설정. 봇 매매와 시각적 일치를 원한다면 차트 설정을 봇과 동일하게 (fast=10, slow=30, base=200) 조정 권장.

## 8. 결론 (최종, 이미지 + DB 종합)

### 결론 #1: 시각적 Cross ≠ 봇 Cross
- 04:01에 차트에서 두 라인이 분리되는 모습은 **gap이 +0.069(거의 0) → +3.868로 급격히 벌어진 시각적 결과**
- 봉 종가 기준으로 fast는 단 한 번도 slow 아래로 떨어지지 않음 → **봇 Cross 조건 미충족**

### 결론 #2: 차트 EMA와 봇 EMA의 기간 불일치 가능성 (확인 필요)
- 차트 인디케이터 표시: `20 200 10` 추정 → 노란선이 **EMA 20 (또는 25)**일 가능성
- 봇 설정: fast=10, slow=**30**
- → 차트에서 보이는 Cross 시점과 봇이 인식하는 Cross 시점이 다를 수 있음
- **사용자 액션 권장**: 차트 앱 EMA 설정을 (10, 30, 200)으로 조정하면 봇과 시각 일치

### 결론 #3: 봇은 정상 동작
- 매도(03:21 STOP_LOSS): 설정값 0.5% 손절 임계값 초과로 정상 발동
- 매수 차단(03:22 ~ 14:50): 봇 Cross 조건을 만족한 봉이 없어서 정상 NO_BUY_SIGNAL
- 매수(14:52): 14:24 Dead Cross → 14:51 Golden Cross 전환 후 정상 발동

### 결론 #4: 04:01 봇 미감지의 진짜 본질
- **봇 EMA 30이 EMA 20보다 느리게 반응**
- 매도 직전 가격 하락 시 EMA 30은 거의 흔들리지 않음 (04:00 slow=1904.57)
- 매도 후 가격 회복 시 EMA 10은 +1.05% 양봉 한 번에 점프 가능
- 그러나 EMA 30은 천천히 따라옴 → fast가 slow를 잠시도 밑돌지 못함
- → **만약 봇이 EMA 20을 slow로 사용했다면 매도 직전~04:00 사이에 Dead Cross 발생 가능, 04:01에 Golden Cross 발생 가능**

### 결론 #5: 비싼 재진입 (구조적 손실)
- 매도 1,898 → 재매수 2,008 = **+5.79% 비싼 가격**
- 누적 손실 효과 ≈ -6.6%
- EMA 30의 "느린 반응" + Cross 정의의 "엄격함"이 결합된 결과
