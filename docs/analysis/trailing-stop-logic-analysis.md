#### **STEP 3: 신고가 추적**

**코드**: `core/position_state.py:174-191`
```python
def update_highest_price(self, current_price: float):
    # trailing_armed == True일 때만 작동
    if not self.trailing_armed:
        return

    if self.highest_price is None or current_price > self.highest_price:
        self.highest_price = current_price
```

**특징**:
- 매 봉마다 신고가 갱신 (상승 시에만)
- 하락 시에는 갱신 안 됨 (최고점 유지)

#### **STEP 4: 매도 조건 체크**

**코드**: `core/position_state.py:193-233`
```python
def arm_trailing_stop(self, threshold_pct: float, current_price: float) -> bool:
    max_profit = self.highest_price - self.avg_price       # 최대 수익 금액
    profit_drop = self.highest_price - current_price       # 수익 손실 금액
    profit_drop_pct = profit_drop / max_profit             # 수익 손실률

    if profit_drop_pct >= threshold_pct:
        return True  # 매도 신호

    return False
```

### 1.3 핵심 공식

**매도 조건**:
```
(신고가 - 현재가) / (신고가 - 진입가) >= trailing_stop_pct
```

**계산 예시**:

| 진입가 | 신고가 | 현재가 | 최대 수익 | 수익 손실 | 수익 손실률 | 10% 기준 매도 |
|--------|--------|--------|-----------|-----------|------------|--------------|
| 100원 | 120원 | 119원 | 20원 | 1원 | 5% | ❌ 유지 |
| 100원 | 120원 | 118원 | 20원 | 2원 | 10% | ✅ 매도 |
| 100원 | 120원 | 113원 | 20원 | 7원 | 35% | ✅ 매도 |
| 100원 | 105원 | 104.5원 | 5원 | 0.5원 | 10% | ✅ 매도 |

---

### 1.4 절대 하락률 vs 수익 손실률 비교

**사용자 이해**: "신고가 대비 10% 하락 시 매도"

**실제 코드**: "벌어들인 수익의 10% 손실 시 매도"

**차이점**:
- **절대 하락률**: 신고가 120원 → 108원 (12원 하락 = 10%)
- **수익 손실률**: 신고가 120원, 진입가 100원 → 118원 (수익 20원 중 2원 손실 = 10%)

**결론**: 실제 매도는 **절대 하락률보다 훨씬 빨리** 발생

## 3. 설정별 비교 분석

### 3.1 Trailing Stop 비율별 특징

#### **10% 설정 (보수적)**

**장점**:
- 수익 보호 강력 (벌어들인 수익의 90% 보장)
- 급락 시 빠른 대응
- 심리적 안정감

**단점**:
- 작은 조정에도 청산 가능
- 변동성 장에서 조기 청산 빈번
- 큰 수익 기회 놓칠 수 있음

**예시**: 진입 100원 → 신고가 120원 → **118원 매도** (+18% 수익)

---

#### **50% 설정 (공격적)**

**장점**:
- 장기 추세 추종 (큰 수익 추구)
- 일시적 조정 허용
- 변동성 장에서 유리

**단점**:
- 큰 손실 허용 (벌어들인 수익의 절반 반납)
- 추세 전환 포착 늦음
- 심리적 부담 (최고점 대비 큰 하락)

**예시**: 진입 100원 → 신고가 120원 → **110원 매도** (+10% 수익)

## 4. 주요 특징 및 주의사항

### 4.1 핵심 특징

✅ **Take Profit 도달 후 자동 전환**
- 수동 설정 불필요
- Take Profit 값 자동 참조 (`trailing_stop_activation_pct = self.take_profit`)

✅ **수익 보호 메커니즘**
- 벌어들인 수익의 N%만 손실 허용
- 절대 하락률이 아닌 수익 손실률 기반

✅ **Dead Cross 없이도 추세 이탈 감지**
- EMA 크로스 대기 없이 자동 청산
- 손실 확대 방지

---

### 4.2 주의사항

⚠️ **급등 후 급락 시 민감**
- 신고가 갱신 직후 하락 시 즉시 매도 가능
- 예: 신고가 120원 → 118원만 되어도 10% 설정에서 매도

⚠️ **작은 수익에서 활성화 시 조기 매도**
- Take Profit 1.1% 설정 시 매우 작은 수익에서 활성화
- 이후 작은 하락에도 청산
- 예: 진입 1000원 → 1012원 활성화 → 1006원 매도 (+0.6%)

⚠️ **변동성 장 대응**
- 10% 설정: 변동성에 민감 (조기 청산 빈번)
- 50% 설정: 변동성 허용 (장기 보유)

---

### 4.3 필터 우선순위

매도 필터는 다음 순서로 체크됩니다:

1. **Stop Loss** (손절) - 최우선
2. **Dead Cross** (추세 전환) - 2순위
3. **Take Profit** (익절) - 3순위 (Trailing Stop 활성화 시 스킵)
4. **Trailing Stop** (수익 보호) - 4순위
5. **Stale Position** (정체 포지션) - 최후순위

**참고**: `core/filters/base.py` FilterCategory 정의
