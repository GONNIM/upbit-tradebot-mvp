## 📋 Executive Summary

**핵심 문제**:
- WARMUP 이후 실시간 feed 데이터가 **임시 종가**를 포함할 수 있음
- 이는 **EMA 계산 부정확** → **매수/매도 신호 오류** → **수익률 하락**으로 직결

**즉시 조치**:
1. REST API 호출 시 `to` 파라미터 제거 (첫 번째 batch만)
2. 종가 Validation 메커니즘 추가
3. 백테스팅으로 수익 영향 정량화

**예상 효과**:
- 종가 정확도: 현재 미확인 → **100% 확정 종가**
- 전략 성능: 현재 대비 **+5~15% 수익률 개선** (추정)

## 🔍 1. 현재 시스템 아키텍처 분석

### **WARMUP Phase (초기 200개 봉)**

```
┌─────────────────────────────────────────────────────────────┐
│ live_loop.py:549-553                                        │
│                                                             │
│ safe_fetch_rest(                                           │
│     end_ts=now_utc(),      ← ⚠️ 현재 시각                   │

│     ↓                                                      │

│     count=200                                              │
│ )                                                          │
└─────────────────────────────────────────────────────────────┘
```

**문제**:
- `to` 파라미터로 현재 시각 지정
- Upbit API는 **그 시각에 가용한 데이터** 반환
- 봉 확정 후 20초 대기해도 **API 반영 지연** 존재
- 결과: **임시 종가** 또는 **이전 봉까지만** 반환

---

### **실시간 Feed Phase (WARMUP 이후)**

```
매 분 00초 (Clock)
    ↓
⏰ 봉 확정 감지 (16:05:00)
    ↓
⏳ 20초 Jitter 대기 (API finalization)
    ↓
🔄 REST Reconcile
    ↓
safe_fetch_rest(
    end_ts=now_utc(),      ← ⚠️ 16:05:20
    total_count=400
)
    ↓
pyupbit.get_ohlcv(
    to="2026-03-14 16:05:20",  ← ⚠️ 문제!
    count=400
)
    ↓
⚠️ 16:05:00 봉이 아직 API에 완전 반영 안 됨
    ↓
❌ 임시 종가 또는 16:04:00까지만 반환
```

**실제 영향**:
- 매 분봉마다 **임시 종가** 가능성
- EMA 계산에 **누적 오차** 발생
- Golden/Dead Cross **타이밍 오류**

## 💰 2. 수익성 영향 분석

### **시나리오 A: 임시 종가로 인한 오매수**

```
실제 시장 (확정 종가):
  16:05:00 | Close=2950 (하락)
  EMA_fast=2960, EMA_slow=2970 → Dead Cross 진행 중 ❌ 매수 금지

봇이 본 데이터 (임시 종가):
  16:05:00 | Close=2980 (임시 고가)
  EMA_fast=2965, EMA_slow=2970 → Golden Cross로 오판 ✅ 잘못된 매수!

결과:
  - 고점 매수 → 즉시 손실 진입
  - Stop Loss 발동 → -3% 손실
```

**손실 예시**:
- 투자금: 100만원
- 오매수 1회: -3만원
- 월 10회 오류 → -30만원 (30% 손실)

---

### **시나리오 B: 임시 종가로 인한 오매도**

```
실제 시장 (확정 종가):
  16:10:00 | Close=3050 (상승 지속)
  EMA_fast=3020, EMA_slow=3000 → Golden Cross 유지 ✅ 보유

봇이 본 데이터 (임시 종가):
  16:10:00 | Close=3010 (임시 저가)
  EMA_fast=3005, EMA_slow=3000 → Dead Cross로 오판 ❌ 잘못된 매도!

결과:
  - 조기 매도 → 추가 상승분 놓침
  - 기회비용: +5% → 0%
```

**기회비용**:
- 투자금: 100만원
- 놓친 수익 1회: +5만원
- 월 5회 오류 → -25만원 기회비용

### **방안 2: Validation 메커니즘 추가**

**개념**:
```python
# REST 조회 후 검증
rest_df = pyupbit.get_ohlcv(...)

# 최신 봉이 closed_ts와 일치하는지 확인
if rest_df.index[-1] < closed_ts:
    # 재조회 (최대 5회)
    logger.warning(f"최신 봉 누락 감지 | 기대={closed_ts} 실제={rest_df.index[-1]}")
    retry_df = pyupbit.get_ohlcv(count=10)  # to 없이 재조회

    if retry_df.index[-1] == closed_ts:
        # 성공: 최신 봉 추출
        rest_df.loc[closed_ts] = retry_df.loc[closed_ts]
    else:
        # 실패: 봉 스킵 또는 fallback
```

**효과**:
- ✅ 방어적 코딩 (안전성 증대)
- ✅ 방안 1과 병행 가능

**리스크**:
- ⚠️ 재시도로 인한 지연 (최대 30초)
